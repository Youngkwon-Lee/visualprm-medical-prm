기준일: 2026-04-09 · 텍스트+RAG 경로 중심 + VLM 경로(쿼터 이슈 포함)

1) 사용자 화면에서 답이 보이기까지

① Frontend (app.html/app.js)
사용자가 문제/옵션 선택
+ (선택) image_url 전달
→
② API Backend
/agent-answer 또는 /generate-steps 요청 수신
→
③ 응답 렌더링
최종 답/근거/step trace를 UI에 표시

2) /agent-answer 내부 파이프라인 (현재 정상)

Route
정책 생성 (dataset/case_type/question_type)
specialist 선택
→
Retrieve + Rerank
qdrant(memory) 검색 → reranker 정렬
+ document RAG hits 결합
→
Reason + Judge
steps 생성/검증, final_answer_index 생성
retrieval/document 근거 포함 반환

3) /generate-steps 경로 (VLM 생성)

입력
question + options + (image_url/data URL)
→
Provider 호출
Gemini/OpenAI 멀티모달 생성 호출
→
현재 상태
429 quota/cap으로 차단
(생성 경로 일시 불가)
