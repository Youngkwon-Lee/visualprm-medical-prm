요즘 RAG는 어떻게 구축하는가

최신 RAG 구조, 핵심 기능, 의료 멀티모달 사례, 그리고 visualprm에 바로 적용할 선택지를 한 장으로 정리했습니다.

한눈에 보는 결론

최신 RAG는 단순 top-k 검색이 아닙니다. 보통 routing, reranker, adaptive selection, 때로는 agent loop까지 포함합니다.

1. Route 질문의 도메인, modality, task type을 먼저 판별한다.
2. Retrieve dense / sparse / multimodal retriever로 넓게 후보를 가져온다.
3. Rerank 더 적합한 evidence를 다시 정렬한다.
4. Select 고정 top-k 대신 adaptive-k로 필요한 만큼만 context를 선택한다.
5. Generate / Judge reasoner가 근거를 읽고 답하고, 필요하면 critic / judge가 검증한다.
Route domain, modality, task type 분류
→
Retrieve dense / sparse / multimodal 검색
→
Rerank 관련성, 신뢰도, 도메인 적합성 재정렬
→
Select adaptive-k로 필요한 근거만 선택
→
Generate reasoner / judge가 grounded answer 생성

최신 RAG 아키텍처

요즘 기본 골격

  • ingest: 문서, 리포트, KG, 사례 메모리 수집
  • index: dense, sparse, metadata 인덱스를 함께 사용
  • route: 질문에 맞는 검색기를 고름
  • retrieve: 1차 후보를 넓게 가져옴
  • rerank: 더 믿을 만한 근거를 다시 정렬
  • adaptive selection: 필요한 만큼만 context 선택
  • generation: reasoner가 근거를 읽고 답 생성
  • evaluation: accuracy 외에 faithfulness도 평가
대표 참고: Retrieval-Augmented Generation for AI-Generated Content: A Survey (2025), Awesome RAG Papers

요즘 핵심 기능

reranker / adaptive retrieval / agentic RAG

  • Reranker: 검색 결과를 다시 정렬
  • Adaptive retrieval: 상황에 따라 context 개수를 조절
  • Agentic RAG: 필요하면 다시 검색
  • Hybrid retrieval: dense + sparse + metadata + KB를 함께 사용
  • Corrective loop: retrieval이 약하면 query를 다시 만든다
핵심
벡터 DB만으로 끝나지 않습니다. rerank와 adaptive selection이 품질 차이를 만듭니다.
관련 참고: Agentic RAG Survey, Advanced RAG pattern notes

Medical Multimodal RAG 사례

쉽게 말하면, 최근 의료 멀티모달 RAG 논문들은 “비슷한 자료를 찾아서 그냥 붙인다”보다 “어떤 자료를 가져오고, 그중 무엇을 더 믿을지 고른 뒤, 모델이 그 근거를 읽고 답하게 한다” 쪽으로 발전하고 있습니다.

논문 무엇을 retrieval했는가 핵심 아이디어 우리에게 주는 시사점
MMed-RAG
ICLR 2025		 도메인별 external context domain-aware retrieval + adaptive context selection 도메인 라우팅과 adaptive-k가 핵심
FGRW
Information Fusion 2025		 PEIR, ROCO 등 image-text knowledge retrieved knowledge를 다시 가중합 검색 후 선택과 fusion이 중요
MKGF
Neurocomputing 2025		 multi-modal medical knowledge graph graph/entity/image 기반 retrieval case-memory 외에 KG source 가치 큼
MasonNLP at MEDIQA-WV 2025
ClinicalNLP 2025		 in-domain textual + visual examples 간단한 multimodal example RAG baseline case-memory도 가능하지만 selection이 중요
쉽게 정리하면
최근 논문들은 “검색해서 붙이기”에서 끝나지 않습니다. 보통 도메인에 맞게 검색하고, 다시 정렬하고, 그 근거를 바탕으로 모델이 답하도록 만듭니다.

Med-PRM과 비슷한 흐름의 연구

완전히 같은 논문은 많지 않지만, 최근에는 retrieval로 근거를 가져오고, intermediate reasoning step을 검증하거나 보상하는 연구 흐름이 분명히 보입니다.

연구 핵심 방식 Med-PRM과의 관계 우리에게 주는 시사점
Med-PRM
2025		 가이드라인/문헌 retrieval + step verification 기준점 retrieval을 answer generation보다 step 검증에 쓰는 흐름
MedCEG
2025		 critical evidence graph + procedure reward 매우 가까움 step을 graph/evidence 단위로 관리하는 방향 참고 가능
Retrieval-Augmented Process Reward Model
ACL Findings 2025		 retrieval + PRM으로 reasoning 일반화 개념적으로 가까움 medical이 아니어도 PRM과 retrieval 결합은 강한 흐름
Step-wise Verification / Generative Reward 계열
2025		 중간 reasoning step마다 검증/보상 핵심 문제의식이 비슷함 최종 답보다 중간 과정 품질을 보는 방향 강화 가능
RaR: Retrieval and Reasoning for Radiology QA
2025		 retrieval + multi-step reasoning 직접적이진 않지만 관련 높음 radiology 쪽에서 retrieval과 reasoning을 연결하는 실전 예시
쉽게 말하면
요즘 흐름은 “자료를 검색해 답만 잘 맞히자”보다 “자료를 검색해 reasoning 과정이 맞는지 확인하자” 쪽으로 확장되고 있습니다.
참고: Med-PRM, MedCEG, Retrieval-Augmented Process Reward Model, Step-wise Verification, RaR

우리 프로젝트 추천 아키텍처

1. Domain Router Pathology / Radiology / General 분기
2. Hybrid Retrieval case-memory + document + optional KG
3. Medical Reranker modality / question type / domain term 반영
4. Adaptive Context Selection quality 기반 context 선택
5. Reasoner / Critic answer 생성 + step quality 평가

Input Layer

image + question + options + dataset metadata

Routing Layer

specialist와 retrieval policy 결정

Evidence Layer

case-memory + document + optional KG 검색

Selection Layer

medical reranker + adaptive-k로 evidence 정제

Reasoning Layer

reasoner가 근거를 읽고 trace와 answer 생성

Verification Layer

critic / judge가 step quality와 consistency를 확인
