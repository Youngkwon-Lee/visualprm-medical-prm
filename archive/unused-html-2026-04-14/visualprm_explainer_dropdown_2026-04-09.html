VisualPRM 전체 설명 (드롭다운)

필요한 항목만 펼쳐서 확인할 수 있는 버전
1) 현재 요약 Executive
  • 텍스트+RAG 경로는 정상 동작 (/agent-answer 200)
  • VLM 생성 경로는 현재 quota 이슈로 차단 (/generate-steps 429)
  • RunPod는 pod 생성 가능하지만 SSH exec 경로가 불안정
2) 사용자 요청이 답변으로 보이기까지 Data Flow
  1. Frontend(app.html/app.js)에서 question/options(+image_url) 전송
  2. Backend API가 요청 수신 (/agent-answer or /generate-steps)
  3. policy 생성 → specialist 라우팅
  4. qdrant에서 retrieval + reranker 정렬 + document RAG 결합
  5. 최종 답/근거/step 정보를 JSON으로 반환
  6. Frontend에서 trace + 최종답 렌더링
3) specialist는 무엇이 있나? Routing
  • general_medical_reasoner (기본)
  • radiology_reasoner (영상의학)
  • pathology_reasoner (병리)
  • modality_recognition_reasoner (modality 인식형 질문)
분기 기준은 dataset/case_type/modality/question_type/질문 키워드(예: uptake, nucleus, h&e 등)입니다.
4) qdrant는 어떻게 만들었나? RAG Index
  • 컬렉션: medical_vqa_cases
  • 기본 모드: RAG_QDRANT_MODE=memory
  • 벡터 차원: 256
  • 인덱싱 대상: dataset 케이스(question/options/modality/case_type 등)
  • payload 포함: dataset, id, case_type, modality, question, options, image_url
임베딩은 기본 hash-fallback, 옵션으로 sentence-transformers 사용 가능. RAG_AUTO_WARM=1이면 서버 시작 시 자동 warmup 됩니다.
5) Document RAG는 어떤 걸 쓰나? Evidence
현재 로그에서 확인된 대표 예시:
  • Nuclear enlargement: 핵 비대 판독 포인트 설명
  • Hyperchromasia and atypia: H&E 맥락의 비정형 소견 설명
즉, 모델 단독 추론이 아니라 문서 근거를 함께 붙여 답변 품질/설명력을 높입니다.
6) 왜 VLM 생성은 막혀 있나? Quota
  • Gemini: RESOURCE_EXHAUSTED (monthly spending cap 초과)
  • OpenAI: insufficient_quota
코드/로직 문제가 아니라 provider 결제/한도 설정 이슈입니다.
7) RunPod는 왜 안 됐나? Infra
여러 GPU/Cloud 조합에서 pod는 RUNNING이지만, SSH public port가 미할당이거나 connection refused가 반복되었습니다. 즉, 스케줄링은 되지만 exec(SSH) 채널이 불안정한 상태입니다.
8) 지금 당장 가능한 운영 전략 Action
  • 단기: /agent-answer 중심 데모/평가 지속
  • 중기: provider quota 복구 후 VLM 1→2→20 샘플 순차 검증
  • 병행: RunPod 지원 문의(SSH 포트 증빙 첨부)
