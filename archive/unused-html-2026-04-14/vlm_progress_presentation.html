What Changed

text-only에서 실제 VLM 생성 경로로 전환

Before
Qwen2.5-7B-Instruct text-only 모델 사용
질문 / 선택지 / 메타데이터만 보고 rationale 생성
이미지는 메타데이터에만 있고 실제 추론에는 미사용
VisualPRM 논문 방식과 핵심적으로 어긋남
Now
Qwen2.5-VL-7B-Instruct 기반 RunPod 경로 구축
image_url를 backend와 model server까지 실제 전달
qwen-vl-utils, 최신 transformers, torch 2.7.1 반영
PathVQA 20-case와 100-case 샘플 생성까지 확인

핵심 포인트는 "이미지를 넣는 척"이 아니라, 실제로 model server가 이미지 파일을 읽고 답을 생성하도록 만든 것입니다.

Pipeline

현재 데이터 생산 흐름

1

PathVQA case + image bundle 준비

vlm_sample_pathvqa_100 안에 JSON과 실제 이미지 100장을 함께 구성.

2

RunPod에서 VLM server + backend 기동

Qwen2.5-VL-7B-Instruct를 로드하고 /generate-steps로 OpenAI-compatible 호출.

3

100-case rationale generation

각 case에 대해 image + question + options로 step rationale과 final answer 생성.

4

정답 일치 필터 + SFT 포맷 변환

정답 letter가 맞는 샘플만 유지해서 multimodal train / val JSONL 생성.

Assessment

Good News

PathVQA 100개만으로는 style이 안정적이라고 보기 어렵다
바로 대규모 본학습보다 2차 필터 또는 추가 batch가 더 안전하다
