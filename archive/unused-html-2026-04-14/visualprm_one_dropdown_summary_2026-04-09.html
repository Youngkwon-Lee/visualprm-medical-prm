VisualPRM 워크플로우 요약 (원 드롭다운)

요청 항목 5개를 하나의 펼침 섹션으로 통합
펼쳐보기: specialist 역할 + RAG 생성 + qdrant 워크플로우 + 데이터셋 예시 + VLM 차단 원인

1) 에이전트(specialist)별 역할

  • general_medical_reasoner: 기본 fallback 추론
  • radiology_reasoner: 영상의학 질문(CT/MRI/X-ray/uptake/pattern)
  • pathology_reasoner: 병리 질문(nucleus/h&e/histology/atypia)
  • modality_recognition_reasoner: modality 인식형 질문

2) RAG는 어떻게 만들어졌나

  • 데이터셋 케이스(question/options/modality 등)를 텍스트화
  • 임베딩 생성(기본: hash-fallback, 옵션: sentence-transformers)
  • 유사 케이스 검색 + rerank + 문서 근거(document RAG) 결합

3) qdrant 워크플로우

  1. 컬렉션 medical_vqa_cases 생성
  2. 케이스별 벡터와 payload(dataset/id/question/options/image_url) upsert
  3. 질문 입력 시 top-k 검색
  4. 필터 적용(allowed_datasets, exclude_same_case)
  5. reranker 정렬 후 최종 reasoning에 투입

4) 우리 데이터셋 실제 예시

  • 케이스: pathvqa_000000
  • 질문: Is the nucleus enlarged?
  • 흐름: pathvqa 입력 → pathology_reasoner 분기 → qdrant retrieval → 문서근거 2건 결합 → /agent-answer 200

5) 왜 VLM 생성이 막혔나

  • Gemini: RESOURCE_EXHAUSTED (monthly spending cap 초과)
  • OpenAI: insufficient_quota

현재 가능한 것: /agent-answer 기반 텍스트+RAG 데모/검증

현재 막힌 것: /generate-steps 기반 VLM 생성(quota 복구 필요)
