RAG-Gym × VisualPRM Loop 테스트 요약

작성일: 2026-04-15

1) 이번 테스트 목적

  • WSL(오케스트레이터) + MacBook Ollama(모델) 연결 확인
  • actor-critic 루프 코드가 끝까지 도는지 스모크 테스트

2) 실행 환경

  • 오케스트레이터: Desktop WSL
  • 모델 서버: MacBook Ollama
  • Ollama 모델: gemma4:latest (설치 확인됨)
  • 백엔드: api_backend.py (localhost:8764)

3) 실제 테스트 세팅 (스모크)

  • Provider: open_model
  • Generate/Verify model: gemma4:latest
  • Dataset: pathvqa_test_for_app.json
  • max_samples=2, max_iter=1, num_actions=2
  • Critic mode: random (verify_steps 비활성)
  • 상대경로 이미지 URL은 원격 접근 이슈로 임시 비활성

4) 결과

  • [1/2] pathvqa_000000 pred=0 gold=0 correct=True
  • [2/2] pathvqa_000001 pred=1 gold=1 correct=True
  • Accuracy: 1.0 (2샘플 스모크 기준)

결과 파일:

  • experiments/rag_gym_visualprm_loop/outputs/inference_results.json
  • experiments/rag_gym_visualprm_loop/outputs/summary.json

5) 이 결과의 의미

  • 파이프라인(로딩→생성→선택→저장) 동작은 확인됨
  • 아직 verify_steps critic 성능 검증은 안 한 상태
  • 정확도 1.0은 소규모 스모크라 성능 결론으로 사용 불가

6) 다음 단계 (본 실험)

  1. critic.mode = verify_steps로 변경
  2. max_samples=20, num_actions=5로 확대
  3. 비교: random critic vs verify_steps critic
  4. 가능하면 이미지 URL을 HTTP로 제공해 멀티모달 입력 복구
