RAG-Gym 정식 파이프라인 (RAG 포함)

업데이트: 2026-04-16 · 핵심: 반복(Loop)과 검색(RAG)은 다르며, 정식 파이프라인은 둘 다 포함

쉬운 예시

문제: "이 병리 이미지 소견으로 A(악성) / B(양성) 중 무엇인가?"

  1. Actor가 후보 3개 생성
    • 후보1: A, query="특정 핵 비정형 기준"
    • 후보2: B, query 없음
    • 후보3: A, query="유사 케이스 병리 소견"
  2. Critic이 채점: 0.81 / 0.52 / 0.76
  3. 최고점 후보1 선택 (0.81)
  4. 후보1에 query가 있으므로 RAG 검색 수행 → 근거 문서 top-k 확보
  5. 문서를 history에 추가하고 다음 step에서 다시 후보 생성/채점
  6. 최종 step에서 answer가 확정되면 종료

즉: 여러 풀이 생성 → 채점 → 필요 시 검색으로 근거 보강 → 최종 답 확정
