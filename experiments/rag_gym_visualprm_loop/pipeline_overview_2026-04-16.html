RAG-Gym × VisualPRM 파이프라인 정리

업데이트: 2026-04-16

1) 현재 파이프라인 (구현 상태)

  1. 데이터 로드 (pathvqa_test_for_app.json)
  2. 입력 상태 구성 (question/options/case_type/modality)
  3. Actor: /generate-steps로 후보 num_actions개 생성
  4. Critic:
    • random 모드: 랜덤 점수
    • verify_steps 모드: /verify-steps step score 평균
  5. 최고 점수 후보 선택 (argmax)
  6. 필요 시 루프 반복 (max_iter)
  7. 최종 답 저장 + 정답 비교 + 결과 파일 출력

2) 텍스트 기반인가? 이미지 기반인가?

구분현재 상태설명
질문/선지/메타 사용 중 텍스트 입력으로 정상 사용
이미지 입력 활성 (HTTP URL) image_url_prefix=http://100.125.26.99:8766/visualprm로 상대경로를 절대 URL로 변환해 전달
결론 현재는 이미지+텍스트 멀티모달 입력으로 실행 단, 성능 평가는 소샘플(3개) 기준이므로 확장 검증 필요

3) 멀티모달(이미지 포함)로 완전 복구하려면

  1. 이미지를 HTTP URL로 노출 (http://.../visualprm/images/...)
  2. image_url_prefix를 config에 설정
  3. Actor 호출에서 image_url 전달 유지
  4. 성능 비교: 텍스트-only vs 이미지+텍스트

4) 최신 실행 결과 (2026-04-16)

  • 세팅: 샘플 수=3, 문제당 루프 횟수=1, 후보 답안 수=3, critic=verify_steps
  • 모델: gemini-2.5-flash (generate/verify)
  • 결과: 2/3, Accuracy 0.6667

5) 출력 파일

  • experiments/rag_gym_visualprm_loop/outputs/inference_results.json
  • experiments/rag_gym_visualprm_loop/outputs/summary.json
