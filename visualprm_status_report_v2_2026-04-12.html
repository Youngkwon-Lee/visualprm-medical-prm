RAG + PRM Selection 상태 요약

2026-04-12 기준: 안정성은 개선, 정확도/편향 개선은 제한적

Team / Name: Youngkwon · Jarvis
Experiment Design

요청 반영: 3단계 실험 + 2개 진단

1) Baseline → 2) Baseline vs RAG → 3) Baseline vs RAG vs RAG_PRM

1

Baseline (단순 Gemini)

  • 엔드포인트: /generate-steps
  • 입력: question + options (+ image_url)
  • 동작: Gemini가 step 생성 후 final_answer 선택
  • 특징: RAG/PRM 없이 모델 단독 추론
  • 결과: acc 약 0.50, 0번 쏠림
2

RAG 파이프라인 (도식)

  • Input → Route → Retrieve(Qdrant)
  • → Grounding(retrieval_context)
  • → Gemini reasoning → final_answer
  • 엔드포인트: /agent-answer
  • 결과: baseline 대비 정확도 이득 미미
3

RAG + PRM 파이프라인 (도식)

  • Input → RAG → BoN 후보 N개
  • → PRM score(candidate별)
  • → best candidate 선택 → final_answer
  • 모드: selection_mode=prm, bon_n
  • 현재: PRM 선택은 동작하나 후보가 0으로 붕괴
Diag 1

Error Breakdown

  • 10샘플 오답 5건
  • 모두 gold=1, pred=0
  • selection collapse(0-bias)
Diag 2

Candidate Diversity

  • PRM 후보 분해 결과
  • 10/10 케이스 후보 전부 0
  • 후보 생성 단계 병목
